# S3.03-Patterns2
**Description**: Academic Java project focused on design patterns. The repository groups three independent mini projects —`Decorator`, `Observer`, and `ProgressiveInterface`— to practice dynamic composition, object notifications, and guided construction through progressive interfaces.

## 📌 Exercise Statement

This project gathers three design pattern exercises:

- **Decorator**: implementation of a Bubble Tea customization system where a base drink can receive extra ingredients (ice, sugar, tapioca, flavors) without creating subclasses for every possible combination.
- **Observer**: implementation of a stock market change notification system where an observable automatically informs multiple subscribed agencies when the market goes up or down.
- **ProgressiveInterface**: implementation of a restaurant menu construction system using Builder + Fluent API + progressive interfaces to guarantee a valid construction order.

## ✨ Features

### `Decorator`
- Common `BubbleTea` contract with `getDescription()` and `getCost()`.
- Base drinks: `LatteBase`, `MatchaBase`, and `TeaBase`.
- Combinable decorators: `IceDecorator`, `SugarDecorator`, `TapiocaDecorator`, and `FlavorDecorator`.
- Dynamic ingredient composition at runtime.
- Tests to validate bases, individual decorators, and chained combinations.

### `Observer`
- `StockbrokerObservable` observable with dynamic observer subscription and removal.
- `StockBrokerageObserver` contract to decouple publisher and subscribers.
- Concrete observer `StockBrokerage` that receives, builds, and prints notifications.
- Simulation of `UP` and `DOWN` market events.
- Tests for subscription, notification, observer removal, and value updates.

### `ProgressiveInterface`
- Menu system built with `Builder` and `Fluent API`.
- Progressive interfaces to enforce a correct construction flow.
- Menu types: `FullMenu`, `HalfMenu`, and `KidsMenu`.
- Dish name validation and restriction of incompatible options.
- Tests for valid sequences, conditional options, and edge cases.

## 🚀 Installation and Run
1. Clone the repository:

```zsh
git clone <repository-url>
cd S3.03-Patterns2
```

2. Environment variables:

- No `.env` file or external configuration is required to run this project.

3. Run the application:

Compile the project:

```zsh
mvn -q -f "/Applications/XAMPP/xamppfiles/htdocs/proyectos/sprint3/S3.03-Patterns2/pom.xml" -DskipTests compile
```

Run each mini project:

```zsh
cd "/Applications/XAMPP/xamppfiles/htdocs/proyectos/sprint3/S3.03-Patterns2"
java -cp target/classes Decorator.App
java -cp target/classes Observer.App
java -cp target/classes ProgressiveInterface.App
```

Run the full test suite:

```zsh
mvn -q -f "/Applications/XAMPP/xamppfiles/htdocs/proyectos/sprint3/S3.03-Patterns2/pom.xml" test
```

Run specific suites:


## 🧩 Diagrams and Technical Decision Rationale
- **Decorator**: composition is used to add behavior without multiplying subclasses; each ingredient is an independent decorator.
- **Observer**: the sender object is decoupled from receivers through an observer interface and dynamic subscription.
- **ProgressiveInterface**: Builder is combined with progressive interfaces to restrict the creation flow at compile time.
- **SOLID**:
  - `S`: each class has a specific responsibility.
  - `O`: new decorators, observers, or menu types can be added without modifying the existing core.
  - `I`: interfaces are small and focused on their role.
  - `D`: modules depend on abstractions (`BubbleTea`, `StockBrokerageObserver`, flow interfaces).
- **Testability**: each mini project includes separate unit tests to verify functionality and prevent regressions.
